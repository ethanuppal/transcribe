Overview

Transcribe is a cross-platform web app that allows you to slow down and loop sections of an audio file to make transcription easier. You can save and restore your progress to work later on.

Uploading Files

To upload a file, simply click the button indicated at the top of the screen. When you upload a file, even if it is the same file as before, all your progress will be cleared, so make sure to save.

Uploading a file will, as said above, generate a fresh workspace. The workspace includes the player and the section editor.

Using the Player

There are three buttons in the audio player. The left arrow button goes to the beginning of the audio; the right arrow goes to the end; and the play button starts and stops it.

When in a section, the beginning- and end-jumping buttons will only go to the bounds specified.

Adding the Sections

The following three buttons are the entire interface for the section editor.

The add section button creates a new section at the bottom of the page.

The save section button will copy some text that represents the sections you have made. More on saving in this section.

The load section text is NOT a button, but the small load to the right of the text box is. More on loading in this section.

Modifying the Sections

Here is what all the boxes are for:

  • Enter the name of the section into the Name box.
  • Enter the start time of the section into the Start box, following a MM:SS (minute:second) timestamp format.
  • Enter the start time of the section into the End box, following a MM:SS timestamp format.
  • Enter the speed of the section into the Speed box.
  • Check or uncheck the Loop box to toggle looping.

Press the trash button at the top-right of a section to irreversibly delete a section.

Saving your Progress

This website does not store any data—audio files or sections—on servers or locally. Therefore, you must take initiative to save your progress.

Press the save button to copy the "section code" to your clipboard. This bizarre-looking string of letters is what encodes all the information needed to bring back your progress.

This section code does NOT have your audio file, so you must keep that as well. I recommend keeping the audio files in a cloud provider such as Google Drive or (safer) an E2E version and the section codes in any cloud text editor such as Google Docs or (safer) the notes app on your device if it has one.

Advanced

This section is only for advanced users.

The sections are stored as a JSON object like so:

{
    "filename" : Name of file,
    "fileModified" : JS timestamp when file was last modified,
    "lastID" : Number greater than or equal to any other section id,
    "currentTime" : The position of the player,
    "sections" : [
        {
            "id" : Unique id,
            "name" : Name of section,
            "start" : Start time in seconds,
            "end" : End time in seconds,
            "speed" : Speed where 1 means original,
            "loop" : Boolean
        }
    ]
}

You can reorder the way the sections appear in the app by reordering the corresponding JSON objects in the sections array.
